Learn Git

Basic Commands

Git is a distributed version control system. Here are some basic commands to get you started:

  • git status: Show the working directory status.
  • git add [file]: Add a file to the staging area.
  • git commit -m "message": Commit changes with a message.
  • git push: Push changes to the remote repository.
  • git pull: Pull changes from the remote repository.
  • git clone [url]: Clone a repository into a new directory.
  • git init: Create an empty Git repository or reinitialize an existing one.
  • git log: Show the commit logs.

Branching

Branches in Git allow you to work on different versions of a project simultaneously. Here are some commands related to branching:

  • git branch: List all branches.
  • git branch [branch-name]: Create a new branch.
  • git checkout [branch-name]: Switch to a branch.
  • git checkout -b [branch-name]: Create and switch to a new branch.
  • git branch -d [branch-name]: Delete a branch.
  • git branch -D [branch-name]: Force delete a branch.

Merging

Merging is the process of combining changes from different branches. Here are some commands related to merging:

  • git merge [branch-name]: Merge the specified branch into the current branch.
  • git rebase [branch-name]: Reapply commits on top of another base tip.
  • git merge --abort: Abort the merge process and try to reconstruct the pre-merge state.
  • git rebase --abort: Abort the rebase process and return to the pre-rebase state.

Showcase: Merging New Features into Master

When working on a new feature, you typically create a new branch from the master branch. Once the feature is complete, you merge it back into the master branch. Here are the steps:

  1. Switch to the master branch: git checkout master
  2. Pull the latest changes: git pull
  3. Switch to your feature branch: git checkout [feature-branch]
  4. Merge the master branch into your feature branch to resolve any conflicts: git merge master
  5. Switch back to the master branch: git checkout master
  6. Merge your feature branch into the master branch: git merge [feature-branch]
  7. Push the changes to the remote repository: git push
